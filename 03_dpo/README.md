# 03 - DPO（直接偏好优化）

> 不需要奖励模型，直接用偏好数据优化模型。

## 从 RLHF 到 DPO 的演化

### RLHF（传统方式，复杂）

```
Step 1: SFT 微调基座模型
Step 2: 收集人类偏好数据（对同一问题的两个回答排序）
Step 3: 训练一个奖励模型（Reward Model）
Step 4: 用 PPO 算法做强化学习（最大化奖励模型的打分）
```

问题：步骤多、需要训练额外模型、PPO 训练不稳定、超参敏感。

### DPO（简化方式）

```
Step 1: SFT 微调基座模型
Step 2: 收集偏好数据 ← 和 RLHF 一样
Step 3: 直接用偏好数据训练 ← 不需要奖励模型和 PPO！
```

DPO 论文的核心发现：**可以从理论上推导出一个等价的损失函数，直接用偏好数据优化模型，跳过奖励模型训练和 PPO。**

## DPO 损失函数

```
L_DPO = -log σ(β × (log π(y_w|x)/π_ref(y_w|x) - log π(y_l|x)/π_ref(y_l|x)))
```

直觉理解：
- `y_w`：人类偏好的回答（chosen/winner）
- `y_l`：人类不偏好的回答（rejected/loser）
- `π`：当前模型
- `π_ref`：参考模型（通常是 SFT 后的模型）
- `β`：温度参数，控制偏离参考模型的程度

目标：让模型给好回答更高的概率，给差回答更低的概率，同时不要偏离参考模型太远（防止 reward hacking）。

## 数据格式

DPO 需要**偏好数据对**：

```json
{
  "prompt": "什么是人工智能？",
  "chosen": "人工智能（AI）是...",    // 好回答
  "rejected": "AI就是机器人啊..."     // 差回答
}
```

## 运行方式

```bash
source venv/bin/activate
python 03_dpo/train_dpo.py
```
