# 02 - LoRA（低秩适应）

> 用 1% 的参数量达到接近全量微调的效果。

## 什么是 LoRA？

LoRA（Low-Rank Adaptation）是一种**参数高效微调**方法。核心思想是：

> 模型微调时，权重的变化量（ΔW）是**低秩**的，可以用两个小矩阵的乘积来近似。

### 数学原理

全量微调：`W' = W + ΔW`（需要更新整个 W，显存爆炸）

LoRA：`W' = W + B × A`

```
原始权重 W:  [d × d]  (例如 4096 × 4096)
LoRA A:     [d × r]  (例如 4096 × 8)   ← 极小！
LoRA B:     [r × d]  (例如 8 × 4096)   ← 极小！
```

其中 `r` 是秩（rank），通常取 4、8、16。

**参数量对比**（以 d=4096, r=8 为例）：
- 全量微调：4096 × 4096 = 16,777,216 参数
- LoRA：4096 × 8 + 8 × 4096 = 65,536 参数（**仅 0.39%**）

### 为什么有效？

微调时权重的变化集中在一个低维子空间中。LoRA 恰好利用了这一点——用两个低秩矩阵捕捉这个子空间内的变化，忽略其余维度。

## 关键概念

### rank（秩）
LoRA 的核心超参数。rank 越大，LoRA 的表达能力越强，但参数量也越多。
- `r=4`: 极致效率，简单任务够用
- `r=8`: 常用默认值，多数场景表现好
- `r=16-64`: 复杂任务可能需要更高秩

### alpha（缩放因子）
控制 LoRA 更新的强度：`ΔW = (alpha/rank) × B × A`

通常设 `alpha = 2 × rank`，这样 `alpha/rank = 2`，相当于把 LoRA 的更新放大 2 倍。

### target_modules（目标模块）
LoRA 不是加在所有层上，而是选择性地加在 attention 层的投影矩阵上：
- `q_proj`, `v_proj`：最常见的选择
- `k_proj`, `o_proj`：也可以加，效果略有提升
