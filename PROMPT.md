# Mondrian Blocks Puzzle - 完整开发需求

## 需求背景
把实体桌游"蒙德里安逻辑方块"做成在线 H5 版本。用户收到链接后在手机浏览器（包括微信内置浏览器）打开即可游玩。

## 技术约束
- **单文件 HTML**：所有 CSS/JS 内联在一个 index.html 中
- **零构建依赖**：不用任何框架，纯 vanilla JS
- **唯一外部依赖**：PeerJS CDN（用于双人对战 P2P 通信）
- **Mobile-first**：触摸拖拽优先，也支持鼠标
- **微信浏览器兼容**：避免使用 webkit 私有特性

## 游戏规则
- 8×8 网格棋盘（64格）
- 11 个矩形方块，总面积 = 64
- 每关题目：3 个黑色固定块预置在棋盘指定位置
- 玩家任务：把剩余 8 个彩色块全部拖入棋盘，完全填满，无缝隙无重叠
- 每关只有一个正解
- 彩色块可以旋转 90°

## 题库格式
每关一个 JSON 对象：
```json
{
  "id": 1,
  "difficulty": 1,  // 1=初级 2=中级 3=高级 4=大师
  "fixed": [
    {"id": "K1", "x": 3, "y": 4},
    {"id": "K2", "x": 1, "y": 2, "rotated": false},
    {"id": "K3", "x": 5, "y": 0, "rotated": true}
  ]
}
```
x,y 都是 0-7 的网格坐标（左上角为 0,0）。rotated=true 表示旋转90°（宽高互换）。

## 题库生成
需要内置一个 solver（暴力搜索），用来：
1. 验证每关题目确实有且仅有一个解
2. 生成足够多的关卡（目标至少 50 关，按难度分级）

Solver 逻辑：给定 3 个黑块位置，尝试把 8 个彩色块（每个可旋转）填入棋盘，用回溯法搜索。

**难度判定**：solver 的回溯次数越多，难度越高。

## 功能模块

### 1. 主界面
- 顶部：标题 "MONDRIAN" + 当前关卡信息
- 中间：8×8 棋盘
- 下方：方块托盘（待放置的彩色块）
- 底部：功能按钮栏

### 2. 棋盘
- CSS Grid 渲染 8×8 网格
- 黑色粗边框（5px #111）
- 网格线（3px #111 gap）
- 空格背景：#EBE6D8

### 3. 方块托盘
- 水平排列可拖拽方块
- 每个方块显示为小型 grid 预览
- 可滚动（方块多时）

### 4. 拖拽交互
- **触摸**: touchstart/touchmove/touchend
- **鼠标**: mousedown/mousemove/mouseup
- 拖拽时显示半透明 ghost 跟随手指
- 接近网格时自动吸附（snap to grid）
- 合法位置：绿色高亮
- 非法位置（重叠/越界）：红色高亮
- 松手时：合法→放置，非法→弹回托盘

### 5. 方块旋转
- 方块在托盘中：点击/tap 旋转 90°
- 方块在棋盘上：双击/double-tap 旋转
- 旋转动画（CSS transform transition 200ms）

### 6. 关卡选择器
- 网格布局显示所有关卡
- 按难度分 Tab（初级/中级/高级/大师）
- 已通关显示 ✅ 绿色
- 当前关高亮
- 锁定机制：每级通过 50% 才解锁下一级

### 7. 计时器
- 开始拖第一个方块时启动
- 显示 MM:SS 格式
- 通关时记录最佳时间

### 8. 撤销/重置
- 撤销：回退上一步操作
- 重置：清空棋盘所有彩色块回到托盘

### 9. 完成检测 + 庆祝
- 每次放置后检查：棋盘是否完全填满
- 全部填满 → 胜利！
- 庆祝动画：蒙德里安风格的色块飞散+汇聚

### 10. 双人对战（PeerJS）
引入 PeerJS CDN：
```html
<script src="https://unpkg.com/peerjs@1.5.4/dist/peerjs.min.js"></script>
```

流程：
1. 点「邀请好友对战」→ 创建 Peer，获得随机 peer ID
2. 生成邀请链接：`当前URL?battle=<peerID>&level=<levelID>`
3. 好友打开链接 → 创建 Peer → 连接到房主 peer
4. 双方加载同一关卡
5. 倒计时 3-2-1 开始
6. 实时同步数据（每次放置/移除方块时发送）：
   - 已放置方块数量
   - 完成百分比（已填格数/需填格数）
   - 是否已完成
7. 对方进度显示在界面顶部（进度条 + 百分比）
8. 某方完成 → 发送完成信号 → 双方显示结果
9. 结果页：谁赢了 + 双方用时

#### 技术栈
- 纯 HTML5 + CSS3 + Vanilla JavaScript（单文件，零构建依赖）
- CSS Grid（棋盘渲染）
- Touch Events + Pointer Events（拖拽交互）
- PeerJS / WebRTC（双人对战 P2P 通信，无需后端服务器）
- localStorage（通关记录持久化）
- CSS backdrop-filter（Apple 毛玻璃质感）

#### 开发过程
- 从 GitHub 开源项目 PureMondrian（PureBasic）和 Mondrian Solver（TypeScript）提取了方块精确尺寸
- 总共 11 个方块：3 个黑色固定块（1×1, 2×1, 3×1）+ 8 个彩色可放置块
- 面积总和 64 = 8×8，完美填满
- 题库通过回溯法 solver 自动生成并验证唯一解
- 双人对战使用 PeerJS P2P，避免了自建 WebSocket 服务器的成本
- UI 参考蒙德里安《红黄蓝构成》经典配色 + Apple Liquid Glass 设计语言

#### 踩坑记录
- GitHub Pages 首次部署 404：需要在仓库 Settings 开启 Pages 并选择正确分支
- 微信内置浏览器不支持部分 CSS 特性：用 fallback 方案
- PeerJS 默认信令服务器在国外，中国可能慢：加了重连和超时处理
- 触摸拖拽在移动端需要 preventDefault 防止页面滚动

## 视觉设计

### 蒙德里安经典配色
- 红 #CC2A2A
- 蓝 #1A3A8A  
- 黄 #E8C820
- 白 #F5F0E0
- 黑 #222
- 灰 #DDD
- 棋盘背景 #EBE6D8
- 页面背景 #F8F4EC

### Apple 质感
- 圆角：方块 8px，按钮 12px，面板 16px
- 毛玻璃面板：`backdrop-filter: blur(20px); background: rgba(255,255,255,0.7);`
- 微妙阴影：`box-shadow: 0 8px 32px rgba(0,0,0,0.08);`
- 方块渐变高光：上方轻微亮色 → 下方实色（类似 iOS 图标）
- 字体：-apple-system, 'SF Pro Display', 'PingFang SC', sans-serif
- 动画：300ms ease-out，弹簧吸附效果
- 暗色/亮色双模式（media query prefers-color-scheme）

### 响应式
- 移动端（<480px）：棋盘宽度 = 95vw，方块托盘水平滚动
- 平板（480-768px）：棋盘居中，托盘换行
- 桌面（>768px）：棋盘固定 400px，托盘在右侧
